# Текущий статус

Этап 0 и Этап 1 (Core) завершены.

EnterpriseOS уже имеет рабочее ядро:

- FastAPI подключён к PostgreSQL.
- Alembic настроен, миграции применяются.
- Frontend на React + TypeScript опубликован на `https://dostavka89.ru`.
- Реализованы маршрутизация, API-клиент и reverse proxy `/api`.
- Реализованы JWT-авторизация, Argon2-хеширование паролей и сессия.
- Создан первый администратор `moonwarior`.
- Реализовано управление пользователями: список, создание, редактирование, смена пароля, права администратора, блокировка.
- Реализован минималистичный Dashboard со статусом системы.
- Реализованы верхняя панель, меню-гамбургер и защищённая навигация.
- Настроено ежедневное резервное копирование PostgreSQL на HDD в `D:\EnterpriseOS\backups\postgresql`.
- Резервные копии хранятся 30 дней.

Визуальное направление зафиксировано:

- светлый воздушный интерфейс;
- мягкие размытые цветовые пятна;
- компактная верхняя панель;
- плавающее меню в стиле macOS;
- крупная единая рабочая поверхность;
- спокойная типографика без перегруженности;
- цветовые акценты только для важных статусов;
- пустой Dashboard, когда от пользователя ничего не требуется.

Текущий этап:

**Этап 2 — Automation Core и локальный n8n.**

Зафиксировано архитектурное решение:

- EnterpriseOS является управляющим ядром и единственным источником истины.
- Бизнес-логика и расчеты выполняются в EnterpriseOS.
- n8n является скрытым исполнительным оркестратором.
- n8n подключается через заменяемый интерфейс `AutomationProvider`.
- Обычные пользователи и администраторы не работают с интерфейсом n8n.
- Типовая автоматизация настраивается через понятный шаблон не более чем за четыре действия.
- Для коммерческой поставки предполагается отдельный self-hosted экземпляр n8n, принадлежащий клиенту.
- Перед первой коммерческой поставкой лицензионная модель проходит письменное согласование с правообладателем и юридическую проверку.

Целевая production-топология:

- EnterpriseOS и предназначенный для него n8n должны работать локально на основном сервере компании.
- `N8nProvider` должен обращаться к локальному экземпляру n8n.
- NL VPS с существующим n8n остается временным отдельным legacy-контуром для Telegram-бота, прокси и уже работающих процессов.
- Новые критические процессы EnterpriseOS не должны зависеть от NL VPS.
- После переноса внутренних чатов и процессов в EnterpriseOS NL VPS планируется отключить.
- Даже локальный n8n взаимодействует с EnterpriseOS только через API и webhook и не получает прямого доступа к PostgreSQL EnterpriseOS.

Подготовка переноса инфраструктуры на Ubuntu сознательно отложена до финальной подготовки к production.

# Правило разработки

Мы не пытаемся написать идеальную систему сразу.

Каждый модуль должен быть закончен прежде, чем начинается следующий.

Если по ходу разработки появляется более удачное решение — архитектура может быть изменена.

EnterpriseOS развивается эволюционно.
